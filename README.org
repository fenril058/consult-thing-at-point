* consult-things-at-point
~consult-things-at-point~ is a small extension for Consult which seeds Consult
commands with the “thing” at point as INITIAL input.

It allows commands such as ~consult-line~ or ~consult-ripgrep~ to start with
a symbol, word, or active region near point automatically filled in.

** Features
- Seed Consult commands with thing-at-point as INITIAL input
- Customizable priority order of thing types (e.g. region → symbol → word)
- Optional per-command overrides
- Preserves the original Consult command interface and interactive behavior

* Installation
** Manual installation
1. Put ~consult-things-at-point.el~ somewhere in your load-path
2. Add the following to your init file

#+begin_src emacs-lisp
(require 'consult-things-at-point)
#+end_src

* Usage
** Commands
- ~consult-line-thing-at-point~
  Run ~consult-line~ with the thing at point used as INITIAL input.

- ~consult-line-multi-thing-at-point~
  Run ~consult-line-multi~ with the thing at point used as INITIAL input.
  The QUERY argument is passed through exactly as in ~consult-line-multi~
  (including prefix arguments).

- ~consult-ripgrep-thing-at-point~
  Run ~consult-ripgrep~ with the thing at point used as INITIAL input.
  The DIR argument follows the original ~consult-ripgrep~ convention and
  receives the raw prefix argument (~interactive "P"~).

** Example key bindings
#+begin_src emacs-lisp
(global-set-key (kbd "M-s l") #'consult-line-thing-at-point)
(global-set-key (kbd "M-s M-l") #'consult-line-multi-thing-at-point)
(global-set-key (kbd "M-s r") #'consult-ripgrep-thing-at-point)
#+end_src

* Customization
** Default thing types
~consult-things-at-point-types~ defines the default list of thing types tried
at point, in priority order.

#+begin_src emacs-lisp
(setq consult-things-at-point-types '(region symbol word))
#+end_src

- Each element must be accepted by ~bounds-of-thing-at-point~
- The first matching thing is used as INITIAL input

** Per-command overrides (optional)
Different Consult commands may benefit from different priorities.  This can
be configured via ~consult-things-at-point-types-alist~.

#+begin_src emacs-lisp
(setq consult-things-at-point-types-alist
      '((consult-line . (region symbol word))
        (consult-line-multi . (region symbol word))
        (consult-ripgrep . (symbol word))))
#+end_src

* How it works
- Try each configured thing type using ~bounds-of-thing-at-point~
- Extract the corresponding buffer substring
- Pass it as INITIAL input to the Consult command
- If no thing is found, run the original Consult command unchanged

* Notes
- This package assumes that the Consult command accepts INITIAL as its last
  argument (as is the case for ~consult-line~, ~consult-line-multi~,
  ~consult-ripgrep~, etc.).
- Arguments obtained via ~interactive "P"~ are passed through unchanged for
  compatibility with the original Consult commands.

* Development
** Adding new wrapper commands
- Preserve the original interactive behavior of the Consult command
- Pass all original arguments through unchanged
- Append INITIAL input as the last argument
- Use the target command symbol (e.g. ~#'consult-ripgrep~) to resolve thing
  type priorities
